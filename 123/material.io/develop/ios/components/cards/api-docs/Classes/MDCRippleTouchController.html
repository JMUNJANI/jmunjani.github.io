MDCRippleTouchController

@interface MDCRippleTouchController : NSObject <UIGestureRecognizerDelegate>

The MDCRippleTouchController is a convenience controller that adds all the needed touch tracking and logic to provide the correct ripple effect based on the user interacting with the view the ripple is added to.

  • view

    A weak reference to the view that the ripple is added to, and that responds to the user events.

    Declaration

    Objective-C

    @property (readonly, nonatomic, nullable) UIView *view;
  • rippleView

    The ripple view added to the view.

    Declaration

    Objective-C

    @property (readonly, strong, nonatomic, nonnull) MDCRippleView *rippleView;
  • delegate

    A delegate to extend the behavior of the touch controller.

    Declaration

    Objective-C

    @property (readwrite, nonatomic, nullable) id<MDCRippleTouchControllerDelegate>
    delegate;
  • gestureRecognizer

    The gesture recognizer used to bind the touch events to the ripple.

    Declaration

    Objective-C

    @property (readonly, strong, nonatomic, nonnull)
    UILongPressGestureRecognizer *gestureRecognizer;
  • shouldProcessRippleWithScrollViewGestures

    If set to NO, the ripple gesture will fail and not be initiated if there are other competing gestures that belong to a UIScrollView. This helps the ripple not be invoked if a user wants to scroll but does so while tapping on a view that incorporates a ripple.

    Defaults to YES.

    Declaration

    Objective-C

    @property (assign, readwrite, nonatomic)
    BOOL shouldProcessRippleWithScrollViewGestures;
  • -initWithView:

    Initializes the controller and adds the initialized ripple view as a subview of the provided view.

    Note: When using this initializer, calling addRippleToView: isn’t needed.

    Declaration

    Objective-C

    - (nonnull instancetype)initWithView:(nonnull UIView *)view;

    Parameters

    view

    The view that responds to the touch events for the ripple. The ripple is added to it as a subview.

    Return Value

    an MDCRippleTouchController instance.

  • -init

    Initializes the controller.

    Note: To effectively use the controller a call to addRippleToView: is needed to provide a view that responds to the touch events for the ripple. The ripple is added to the view as a subview.

    Declaration

    Objective-C

    - (nonnull instancetype)init;

    Return Value

    an MDCRippleTouchController instance.

  • -addRippleToView:

    Adds the ripple view as a subview to the provided view, and adds the ripple’s gesture recognizer to it.

    Note: This needs to be called if using the init initialized rather than the initWithView: initializer.

    Declaration

    Objective-C

    - (void)addRippleToView:(nonnull UIView *)view;

    Parameters

    view

    The view that responds to the touch events for the ripple. The ripple is added to it as a subview.
